    1. Overview of the MTUOC Project

    The MTUOC project, developed at the Arts and Humanities Department of the Universitat Oberta de Catalunya (UOC), is designed to facilitate the training, use, and integration of neural and statistical machine translation (MT) systems, as well as LLMs for translation, in professional translation environments. While most MT toolkits are available under free licenses, their practical use presents several challenges:

    • Technical Skills: Toolkits often require medium-to-high programming knowledge (Python, Bash) and lack comprehensive documentation.
    • Integration: Trained engines cannot be easily integrated into Computer-Assisted Translation (CAT) tools.
    • Hardware Requirements: Training requires powerful GPUs or significant RAM, though simply translating with an already trained model requires far less.

    MTUOC addresses these issues by providing well-documented scripts, graphical interfaces, and a highly configurable server. All project components are distributed under a free GNU GPL license, ensuring professionals and organizations can maintain technological independence and data confidentiality.

    2. Key Components

    The project provides a comprehensive ecosystem of tools covering the entire machine translation lifecycle:

    • Corpus Creation: Tools to convert TMX, SDLTM, and XLIFF files to tabular text. It includes programs for automatic document alignment using Hunalign, as well as comparable corpus mining using SBERT.
    • Corpus Cleaning and Combination: Scripts to remove HTML/XML tags, normalize characters, filter by length, and verify languages using fasttext. An algorith to clean and rescore large parallel corpora is also available. It also includes utilities to expand specialized corpora by retrieving similar segments from large general corpora based on language model perplexity.
    • Preprocessing and Training: Includes tokenizers, truecasers, and subword unit tools like BPE and SentencePiece. It provides training scripts and configuration files for Moses, Marian, OpenNMT, Eole, as well as guides and scripts to fine-tune LLMs for translation.
    • Free Engines: The project distributes pre-trained NMT models for several language pairs and domains.
    • Client programs: A desktop and we application to translate documents is also distributed. We're currently developing a postedition editor.

    3. The MTUOC-server

    The MTUOC-server is a crucial component designed to solve the interoperability gap between machine translation engines and professional CAT tools.

    3.1 Architecture

    The server operates on a client-server model acting as a bridge. The workflow is straightforward:

    • The client (e.g., a CAT tool) sends a segment to the MTUOC-server.
    • The server preprocesses the segment (applying tokenization, truecasing, SentencePiece, etc.).
    • It forwards the processed segment to the underlying MT engine or LLM.
    • Upon receiving the translation, it post-processes the segment (detokenization, undoing subwords, etc.).
    • The final translated text is returned to the client.

    3.2 Protocol Interoperability

    The standout feature of the MTUOC-server is its ability to communicate using multiple protocols, including MTUOC, Moses, ModernMT, OpenNMT, and NMTWizard. The server can mimic these protocols and new protocols can be easily added in the future. For instance, you can use a fast Marian neural engine, but configure the MTUOC-server to behave like a Moses server. This guarantees seamless integration with CAT tools.

    3.3 Supported Systems and LLMs

    The server supports a broad spectrum of translation technologies:

    • NMT Systems: Open-source engines like Marian, OpenNMT, Moses, OpusMT, NLLB, CTranslate2, and Apertium, as well as commercial APIs like DeepL and Google Translate.
    • Large Language Models (LLMs): The server integrates models such as Salamandra, Bloom, ChatGPT, DeepSeek, and Tower. Users can query these via API or by downloading full models from HuggingFace using the transformers library. Furthermore, the server allows users to specify custom translation prompts and use regular expressions to extract the translated text from the LLM's response.

    3.4 Features for Professional Workflows

    To meet professional standards, the server includes several advanced features:

    • XML/HTML Tag Restoration: Neural engines often struggle to place formatting tags correctly (e.g., in DOCX files). The server uses an algorithm that relies on word or subword alignments from the MT system to restore tags. If the MT system does not provide alignments, it can use external fast_align models.
    • Translation Memories: It can integrate translation memories and return a match if the similarity score exceeds a predefined threshold.
    • CAT Tool Plugins: Dedicated plugins are available for direct integration with OmegaT and RWS Trados Studio.

    3.5 Hardware and Platform Compatibility

    The server is multiplatform, running on Linux, Windows, and macOS. While training MT systems requires powerful hardware, running the MTUOC-server for translation does not need a GPU. It can run on standard consumer computers, democratizing access to NMT and LLM technology while allowing translators to keep all client data completely confidential on their local machines.

    4. Starting and stopping a MTUOC-server

    4.0. Getting the MTUOC-server, versions and prerequisites

    To get the MTUOC server go to https://github.com/mtuoc/MTUOC-server. If you have a Python3 interpreter in your system, you can use the Python code and install the prerequisites (requirements.txt). Yo can also go to releases and get the binary version for your operating system.

    4.1. From terminal

    To start a MTUOC-server using the default config-server.yaml config file you can write in terminal:

    python3 MTUOC-server.py

    If you want to use a specific config file (for example config-server-OpusMT.yaml), use:

    python3 MTUOC-server.py config-server-opusmt-eng-spa.yaml

    To stop a MTUOC-server you can write in terminal:

    python3 MTUOC-stop-server.py

    or

    python3 MTUOC-stop-server.py config-server-opusmt-eng-spa.yaml

    A full engine can be downloaded and configured using MTUOC-recipes (take a look at the repository aoliverg/MTUOC-recipes). To download a recipe:

    1. If you already have the recipe as a file:

    python3 MTUOC-downloader.py recipeAina-cat-eng.yaml

    1. Or using the link to the recipe:

    python3 MTUOC-downloader.py https://huggingface.co/datasets/aoliverg/MTUOC-recipes/resolve/main/recipeAina-cat-eng.yaml `

    4.2. Using the MTUOC-server manager

    A GUI application is distributed to make the star/stop/configuration process of MTUOC-servers easier. The application is called MTUOC-server-manager.py. When you start the aplication a GUI interface is shown:

    The application is organized in three tabs:

    • Control: where we can select the engine by its configuration file, start and stop the server and see the status of the server.
    • Test: where we can send sentences to translate.
    • Configuration Editor: where we can edit the basic configuration of the translation engine.
    • Aux. editor: where we can edit the required additional configuration files.
    • Recipes: where you can download full configured engines from recipes.

    4.2.a. The Control tab

    In this tab we can select one of the available server configurations files (these files should be placed in the same directory of the MTUOC-server-manager. We can select one of the available configuration and Start the server. In the console we can observe the relevant information of the server. We can use the Stop button to stop the server.

    4.2.b. The Test tab

    In the Test tab we can write or paste sentences or paragraph to test the started system

    4.2.c. The Configuration Editor tab

    When we click to the button LOAD SELECTED CONFIGURATION & LINKED MODEL CONFIG the configuration are loaded to two tabs: 1. Server Configuration (Main) and 2. Model configuration (Linked). We can observer and make changes to the configuration files and save the changes with the SAVE.... button in the bottom. Please, note that changes are not applied to the running server until we stop and start it again.

    4.2.d. The Aux. Editor tab

    This tab allow for the edition of any text file and it is intended for the edition of additional configuration files needed for some models. You can open any text file with the OPEN button, modify it, and save it with the SAVE AS button.

    4.2.e. The Recipes tab

    This tab allows for the downloading of full engines (including models, alignment models, truecasers, etc.) from a Hugging face repository of MTUOC recipes. By default the "aoliverg/MTUOC-recipes" repository is selected. If you change the repository, use the CONNECT/REFRESH BUTTON to update the information about the available recipes. The recipes can be filtered by a string, as in the example, to limit the number of recipes shown. Select the desired recipe and Donwload it using the DOWNLOAD SELECTED RECIPE button.
