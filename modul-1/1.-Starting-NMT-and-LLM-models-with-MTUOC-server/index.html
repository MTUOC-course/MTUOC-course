    1. Starting NMT and LLM models with MTUOC server

    1. Introduction

    In this unit we will learn to start translations models using the MTUOC-server. To learn what the MTUOC-server is, I recommend reading this paper:

    Antoni Oliver. (2025) MTUOC server: integrating several NMT and LLMs into professional translation workflows. In Proceedings of Machine Translation Summit XX: Volume 2

    Once we start one model, we will also learn how we can actually use these models in our translations. We will explore how to connect a CAT tool to the server and some other ways to get our documents or texts machine translated,

    2. Prerequisites

    Before starting this activity, please check whether the following prerequisites apply to you:

    • How to Download Software from GitHub

    If you work with Windows and you don't plan to do the Units for corpora creation, preprocessing, training and finetuning, you don't need any other prerequisite to follow this Unit,

    However, if you plan to follow the whole course, you will need a running Unix environment. Then:

    • If you work with a Windows computer, follow this: Install Ubuntu on WSL 2
    • If you work with a Mac computer, you can use the Mac terminal, but some installations may be more difficult. You can install Ubuntu with Multipass
    • If you work with a Linux computer, you already have all you need.

    Then, if you don't know how to use the Linux Terminal and the Unix language, follow this tutorial: Fundamental Unix Commands

    3. Get the MTUOC server

    Get the last version of the MTUOC server from Github: https://github.com/mtuoc/MTUOC-server. IMPORTANT: if you plan to run it under Windows (directly, without using Linux in WSL) don't forget to download from releases: MTUOC-server.exe, MTUOC-test-server.exe, marian-server.exe, fast_align.exe, atools.exe.

    Once everything is downloaded, if you use the Windows version directly double click on MTUOC-server.exe and an OpusMT English-Spanish system will be downloaded and started.

    If you are running the Python version under Linux terminal (or Windows with WSL or Mac with Multipass), you need to install the prerequisites (consider whether you want to use a Python virtual environment to install everything or you prefer to use directly the system's Python installation). To install the prerequisites run:

    python3 -m pip install -r requirements.txt

    We should also make sure that the required scripts have executable permissions. Write in the terminal:

    chmod +x fast_align
chmod +x atools
chmod +x marian-server-CPU

    Now, to start the server write into the terminal:

    python3 MTUOC-server.py

    It will start the default model (OpusMT eng-spa, later we will learn how to choose the desired model) and in the terminal something similar to this will appear:

    2026-02-05 16:47:20.370525  Translating with OpusMT models  Helsinki-NLP/opus-mt-en-es
2026-02-05 16:47:20.468116  1   MTUOC server started using MTUOC protocol
MTUOC server IP:    192.168.1.128
MTUOC server port:  8010
MTUOC server type:  MTUOC

    This information shows the started model, the IP, port and server type. All this information is needed in order to the client application (CAT Tool, translation tool, etc.) can connect to this server and get the translations. To test the translation service a simple tool called MTUOC-test-server is distributed with the server. If you're running the server in Windows, you can simply double click on MTUOC-test-server.exe. Otherwise, in terminal write:

    python3 MTUOC-test-server.py

    A simple GUI interface appears allowing you to write a sentence and get the translation:

    In the tab Server you can chose a different IP and Port to connect to a different server. By default these values are set to match with the server we started in the same folder.

    Basic MTUOC-server configuration

    When we start the MTUOC-server writing:

    python3 MTUOC-test-server.py

    the MTUOC-server takes the default configuration file config-server.yaml. We can create as many configuration files as we want and to start the server with a given configuration file (for example config-server-A.txt) we can write:

    python3 MTUOC-test-server.py config-server-A.yaml

    Here we will explain only the basic configuration of the server that can be found in the top lines of the file:

    MTengine: OpusMT
system_name: MTUOC-OpusMT-eng-spa

engine_repository: None

...

MTUOCServer:
  port: 8010
  type: MTUOC
  #one of MTUOC, Moses, ModernMT, OpenNMT, NMTWizard
  verbosity_level: 1
  #specific configuration when acting as ONMT server
  log_file: log.log
    • MTengine: we specify the model we want to use (see 4. Models available in MTUOC-server)
    • system_name: we can set a name to this model. It is not compulsory to set anything special here. The answer from the server in the MTUOC proptocol will include this name.
    • engine_repository: only used when the MTengine is set to download and set the repository from where to download the model.
    • port: the port to use.
    • type: the protocol to use. MTUOC is our own protocol, but some other popular protocols are implemented.
    • verbosity_level: sets the information provided by the server on the screen. It is useful to debug errors.
    • log_file: a file to write the information from the server (related to verbosity_level)
